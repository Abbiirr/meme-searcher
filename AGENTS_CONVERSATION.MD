**Single most important decision (fully evidence-backed):**

Phase 0 retrieval is **caption-then-retrieve**, not naĂŻve cross-modal CLIP. At ingest, Qwen3-VL-8B writes four short labels per image (`caption_literal`, `caption_figurative`, `template_name`, `tags`). Top-1 renders inline in the OWUI chat via markdown image link to a FastAPI thumbnail endpoint.

**Why this shape:**

- 2025 consensus: instruction-tuned / captioned pipelines beat raw CLIP by roughly 60% â†’ 95% on cross-modal retrieval ([TheDataGuy survey](https://thedataguy.pro/blog/2025/12/multimodal-embeddings-evolution/))
- Meme-domain prior art: CM50 (2025) uses VLM-annotated labels exactly like this ([arXiv](https://arxiv.org/html/2501.13851v1))
- MemeCap showed literal caption alone is insufficient for memes â€” need both literal + figurative ([arXiv](https://arxiv.org/abs/2305.13703))
- jina-reranker-v2 is text-only; feeding it reconstructed text documents per image is the supported pattern
- OWUI renders markdown image links reliably; base64 breaks for large images ([issue #16976](https://github.com/open-webui/open-webui/issues/16976))

   - `_call_vlm` (L168â€“169) raises `RuntimeError` when the env var is unset.
   - `caption_image` catches it via the bare-except on L274â€“276 and returns empty `Captions()`.
   - Net effect: every image in a 293-image run ingests "successfully" with `has_caption=False` and empty `retrieval_text`. Qdrant ends up with text-dense vectors derived from raw OCR only, which is a silent retrieval-quality regression â€” BGE-M3 was supposed to embed captions + OCR together.
   - **Mitigation for Codex:** before the first live ingest, confirm `LITELLM_MASTER_KEY` is in the API container's environment (`docker compose exec api env | grep LITELLM`). Consider adding a preflight check in `seed_model_versions()` that logs `ERROR` (not `WARNING`) when the key is missing, so the miss surfaces in CI logs.

2. **Medium â€” 4 sequential calls per image @ 45 s timeout each.**
   - Worst case per image: 3 minutes blocking on gateway hangs.
   - 293 images Ă— worst case = ~15 hours. Realistic throughput is much faster, but one stuck image can stall the whole batch because `ingest_folder` processes sequentially.
   - **Mitigation for Codex:** capture per-image wall-clock in `ops.ingest_steps.meta` during small-ingest. If p99 caption latency > 10 s, consider dropping `CAPTION_TIMEOUT_S` to `20` via env override for the full run so stuck images fail fast.

3. **Medium â€” base64 payload size.**
   - `_image_to_data_url` (L151â€“163) reads the full file and base64-encodes it (~4/3Ă— size). Meme JPEGs are typically < 1 MB, so this is fine â€” but verify LiteLLM's `max_request_size` (defaults vary by version) is at least 10 MB.
   - **Mitigation for Codex:** on the first live call, watch the gateway logs for 413/422 on the VLM alias. If any large PNGs in `data/meme` exceed the limit, pre-downscale to 1024 px max-side before encoding.

4. **Low â€” no retry on transient 5xx from the VLM.**
   - `_call_vlm` calls `raise_for_status()` then surfaces the `HTTPError` up to `caption_image`, which swallows it to empty `Captions`.
   - `infra/litellm/config.yaml::router_settings.num_retries: 2` should cover this at the gateway layer â€” but that only retries LiteLLM's own upstream picks, not client-side timeouts.
   - **Mitigation for Codex:** if small-ingest shows >5% caption-miss rate, add a single client-side retry with 1 s backoff on `HTTPError` with status_code >= 500.

5. **Low â€” smart-quote cleanup missing.**
   - `_clip_sentence` (L216) strips only ASCII `"` and `'`. VLMs frequently emit U+201C/U+201D (`"..."`) and U+2018/U+2019 (`'...'`). Not a correctness bug â€” BGE-M3 tokenizes them fine â€” but worth watching if any template names come back with leading smart quotes that fail the whitelist lookup.

6. **Low â€” four separate TCP connections per image.**
   - `requests.post` without a shared `Session` means 4 Ă— 293 = 1,172 TCP handshakes for the full run. Inside a compose network this is microseconds; over any slower hop it adds up. Not worth refactoring for Phase 0.

7. **Architectural note â€” template prompt drift.**
   - `PROMPT_TEMPLATE` (L68â€“71) tells the VLM "in under 6 words" but VLMs don't always comply. If the VLM emits "this appears to be the drake template meme", `_validate_template` returns `UNKNOWN_TEMPLATE` because the whitelist doesn't contain that phrase.
   - `_validate_template` does one loose-match pass (whitespace-collapse) on L140â€“142, but that won't catch verbose prose.
   - **Mitigation for Codex:** during small-ingest, run `SELECT template_name, COUNT(*) FROM core.image_items GROUP BY 1;` on the 5 rows. If >40% end up as `unknown`, Codex (or Claude in reviewer mode) should harden `_validate_template` with a fuzzy-match over the whitelist (e.g. `rapidfuzz.process.extractOne` at threshold 80) before the full 293-image run.
